How to use API keys

Use your API key in requests with either the Authorization header or X-API-Key header:

# Using Authorization header
curl -H "Authorization: Bearer aiops_your_key_here" http://localhost:5000/api/v1/issues

# Using X-API-Key header
curl -H "X-API-Key: aiops_your_key_here" http://localhost:5000/api/v1/issues

Explore the full API documentation at /api/docs

How to create personal access tokens

GitLab

  1. Visit: https://your-gitlab.com/-/user_settings/personal_access_tokens
  2. Create token with api scope
  3. Copy the token (starts with glpat-)
  4. Add it here using the form above

GitHub

  1. Visit: Settings ‚Üí Developer settings ‚Üí Personal access tokens
  2. Create token (classic) with repo scope
  3. Copy the token (starts with ghp_)
  4. Add it here using the form above

Jira

  1. Visit: https://id.atlassian.com/manage-profile/security/api-tokens
  2. Create API token
  3. Copy the token
  4. Add it here using the form above

Required permissions:

  • GitLab: api scope (full API access)
  • GitHub: repo scope (repository access)
  • Jira: API token with write access to issues

About Global Agent Context

The global agent context allows you to define content that appears in all AGENTS.override.md files, ensuring consistent instructions across all AI agent sessions.

  • Priority: Database global context takes precedence over repository AGENTS.md files
  • CLI Access: Use aiops agents global get/set/clear commands
  • API Access: Available at /api/v1/agents/global
